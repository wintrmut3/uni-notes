Lecture Notes

Introductory Content

A namespace encapsulates all names - functions, variables, classes etc. and isolates them from the rest of the system. std is one such namespace, which holds definitions for cout and endl.

The auto type automatically assigns a type to variables, based on the object being assigned.

continue skips the rest of the code in a loop and goes back to the starting condition check.

break exits the loop entirely. If loops are nested, it exits only the loop in which it was called.

Functions

Functions accept both Pass by Value and Pass by Reference in C++.

Function invocations take as arguments the variables in question (eg. swap(ux,v)), and if in the function definition they are defined as pointers (eg. void swap (int& x, int&y)) the arguments are autocast into reference types, without requiring you to pass pointers directly (though if you want to, you can). If the function definition is : void swap (int x, int y) then arguments are copied into the function during invocation and their copies are modified.

All names must be declared before usage, so we use function prototypes: returnType foo (paramType params);. This is placed at the start of a file, and the declaration can be placed anywhere in the file.

  • The parameter names (params) are dummy vars, and can be set to something else with the same paramType during the actual definition
  • Prototypes are placed before main()

Compilation Basics

C++ compiler takes C++ code and converts to object files. This is the slow step.

C++ linker takes object files (.o) and can combine multiple to generate a binary .exe / .out file.

The compiler can check for:

  • Syntax errors, where the code is misformed (eg. not following the rules of the language structure, missing ;, incomplete braces)
  • Semantic errors, where the code is well formed but contradicts the rules of the language, like setting a string = int , which has undefined meaning

It can neither check for Runtime Errors (eg. dividing by zero or null pointers) nor logical errors (code executes but yields an incorrect result)

The development cycle is about writing code, compiling, testing and debugging until the code is built to spec.

Having all your code in one file is a poor idea, as it leads to longer dev cycles (debugging is much harder, scrollbar is useless) and hinders collaboration so we want separate file compilation.

Separate File Compilation

g++ -E runs only the preprocessor.

g++ -c runs only the compiler.

The #include <systemutil>, #include "user-defined-header.h" and #define name replacer directives simply copy and replace data, without doing any evaluations. The #include directive, for example, copies the contents of the included file and replaces the directive with the pastedump.

In a nutshell, separate-file compilation is a way of writing programs across multiple files.

  1. All functions have a function prototype, defining the function name and arguments. These are stored in individual header files, with the filename functionName.h. These are interface/definition files. Comments are helpful here.
  2. All functions have an implementation file, at the start of which is an include directive to the definition.
// in functionName.cpp implementation
#include functionName.h

... function implementation with constructors/destructor/data members/methods etc
  1. Place appropriate function interface/definition files in any other .cpp file that refers to them, particularly in the c++ file containing the main() function. (Application file)

This way, if we change anything in a file, we need only recompile the files that reference the changed files, and we can link the whole project, including changed and unchanged files, together into a final executable. The make utility tracks the propagation of these changes.

The .h file acts as a contract for contributors (like a C# interface) enforcing the inputs and outputs (ie functionality) of the function.

However, if a file is included twice, or there are global variables/functions they may be declared several times, resulting in errors. There are two ways to prevent this:

  1. #pragma once inside the .h file will make sure it is included only once.
  2. Boolean Guards utilizes the following structure to make sure the file is only included once by defining a unique flag for each header.
#IFNDEF firstInclude
#DEFINE firstInclude
int globalVar;
int globalMethod(params);
#ENDIF

Classes and Objects

In C++, structs are a collection of data, without the ability for access control.

The advantage of Classes is the ability to use access control modifiers.

Classes are data types whose variables are objects (variables with both function and data members). They are defined as follows.

class ClassName{
    //data members
    int x; 
    float y;
    char z;
    //function members
    void SetX(int u); //setter/mutator
    float GetY() //getter/accessor
};

Access modifiers public and private can be used to toggle, until the next public or private, the access control status of all subsequent members.

By default, class members are private.

Private members are not accessible in non-member functions (eg. main()), while public members can be. For example, if we try to print the value of object.privateVar in main, an error is thrown because we are trying to access the private variable inside the main function, which is not a member function of the object's class.

We can typically only access private data members in function member implementations/definitions.

Declaring a variable of a class type is known as instantiation of a class or creating an object.

The Dot (.) operator is used to specify members of objects, while the Scope Resolution Operator ( :: ) is used with class names, rather than object names.

Member functions of a class are often defined in a file called className.h and implemented separately, in className.cpp. The function member definition has the following syntax:

ReturnType ClassName::FunctionName(Paramlist){
    //function definition here!
}

Accessors or Setter Functions are function members that set values of private data members of the same class. They are used to:

  1. Allow error checking when setting values
  2. Prevent direct access/mutation of the data members without knowing where/how
  3. Encapsulation, and hiding the internal state of the object

Every class usually has its own .h file to provide function member prototypes and data member names, and a .cpp file for function member implementation (which includes the class definition header file).

Access control is class based.

Constructors and Destructors

A constructor is a function that is automatically invoked on an object, during its creation.

There can be many constructors, and if one is not defined by the programmer, C++ automatically creates a default constructor with nothing (and zero arguments).

The constructor must have the same name as the class, with no returns. They are usually public but can be private.

While the default constructor has zero arguments, you can create other constructors with different arguments. However, since C++ can only distinguish constructors based on their argument datatypes, each constructor must have a unique order of datatypes. The syntax is as follows:

//prototype in header
 ClassName::ClassName(args);
//implementation
 ClassName::ClassName(args){...}

Constructors can be used to initialize values, though since C++11 (which ecf supports) you can also initialize primitive data members directly in the class definition.

Destructors are called when an object is about to be destroyed, and its memory about to be destroyed. It does not destroy the object, it is only called before destruction. (ie. "Any last words?")

There is only one destructor, with no arguments and no return. There is a tilde before the name of the destructor, which is the class name.

C++ supplies an empty destructor (eg. replace the ellipsis with nothing) if one isn't provided.

//prototype in header
 ClassName::~ClassName();
//implementation
 ClassName::~ClassName(){...}

The assignment operator = is legal for assigning objects (that are initialized) to each other, and it sets complementary data members equal.

The this pointer is a read only variable storing the address of the object that is invoking the function member in which the this is being referenced. The following three statements are therefore equivalent.

className::setValue(v){
    var = v;
    this->var = v;
    (*this).var = v;
}

C++ I/O (<iostream>)

A Stream is a sequence of characters.

cout and cin are intermediaries, interrupting streams going from your code to the screen and from the keyboard into the code respectively. They interpret the stream into and out of the required internal representation.

For example, typing 3 on a keyboard yields the character '3'. Depending on how the code references it:

int val1
char val2
cin >> val1 
    >> val2;

Different interpretations to the stream are given to translate it into the correct datatype.

When asked for input, the OS KBBuffer collects input until the user hits return or EOF, at which point it is sent to cin to deal with.

cin will extract from the stream until the stream is exhausted or a delimiter is met (eg. space). Leading spaces are skipped, and the stream if not consumed is left up to the point of last extraction. cin can be thought of as orthogonal to the stream.

If the stream is exhausted and further input is desired, cin will pause and wait for another stream.

cin will also stop reading if it encounters an invalid input (eg. if reading into an integer, it encounters a '.' or a 'k'). It will fail silently, leaving both the variable being read into, and the stream unaffected. This can lead to an infinite loop if not handled.

cin has a set of boolean flags that are by default, set to false after each extraction operation.

They reflect what happened after the previous extraction operation.

The fail flag triggers if cin fails to extract, for any reason. Note that cin.fail() is used to access the value of the fail flag, and both cin and fail are members of istream.

Since cin might get stuck on a failstate (as when it fails, the stream index cin checks isn't updated) we should change the stream on fail.

cin.clear(); //reset flags to false. otherwise, ignore might fail.
cin.ignore(1000, '\n'); //dumps the first 1000 characters in stream up to the return.

Note that cin.ignore can be used to skip parts of an input of known length (eg. skip the first 3 digits of student ID to get the important parts, or skip up to a space to get the last name from "firstname lastname" string)

There is also a relevant EOF flag that might be raised. This requires the following pattern to be detected correctly:

if(cin.fail()){
    if(cin.eof()){
        //stop taking input, we're done
    }
    else{
        //failed due to another reason:
        cin.clear();
        cin.ignore(1000, ' ');
    }
 else{
    // success / didn't fail
 }
}
